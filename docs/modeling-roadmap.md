# 激光硅掺杂模拟路线图

## 1. 问题定义

我们先建一个“表面有 source 层”的最小模型，用来描述以下过程：

1. 激光在极短时间内向硅表面注入能量。
2. 硅表层升温、熔化，并形成随时间变化的液相层。
3. 掺杂源在表面作为有限库存或边界源，与液相硅发生质量交换。
4. 由于液相中的扩散系数远高于固相，掺杂在熔融层内快速扩散。
5. 冷却和再凝固后，留下最终掺杂分布。

这个问题本质上是一个强瞬态、强非线性、带相变的耦合传热-传质问题。

## 2. 建模建议

### 推荐的起步方式

第一阶段不要直接上完整二维或三维 FEM，而是先做：

- `1D 深度方向模型`
- `瞬态热传导 + 相变 + 掺杂扩散`
- `表面 source 层边界条件`
- `固定几何 + 焓法（enthalpy method）`

原因：

- 激光初版验证阶段最重要的是深度方向上的熔深、熔融时间和掺杂深度。
- 对单点照射或光斑中心附近，前期常可以近似为一维。
- 相变如果先用焓法，不需要显式追踪固液界面，数值实现会稳很多。
- 这样我们能先把物理链条跑通，再扩展到二维轴对称。

### FEM 还是更好的办法

如果目标是最终做成可扩展的多物理场框架，`FEM` 是好选择，尤其适合：

- 温度相关材料参数
- 相变潜热
- 后续扩展到 2D/轴对称/复杂边界

但如果目标是“尽快得到第一个可信结果”，我建议：

- `MVP 用 1D method-of-lines + 有限差分/有限体积`
- `第二阶段再升级到 1D/2D FEM`

原因是这个问题的第一性难点不在网格形状，而在：

- 激光热源建模
- 相变处理
- 扩散系数跨越多个数量级
- source 层如何与液相交换质量

也就是说，先把物理和时间尺度跑顺，比先选最复杂数值框架更重要。

### 我给你的总体建议

建议采用两层路线：

1. `研究模型`：1D，偏快，先验证物理机制和参数敏感性。
2. `生产模型`：2D 轴对称 FEM，加入高斯光斑横向传热和更真实的热源。

## 3. 最小物理模型

### 3.1 热方程

先解温度场：

`rho(T) * cp_eff(T) * dT/dt = d/dz[k(T) * dT/dz] + Q_laser(z, t)`

其中：

- `cp_eff` 可以包含潜热项，等价于焓法
- `Q_laser` 是激光能量沉积项
- 第一版可先假设吸收满足 Beer-Lambert 型深度衰减

### 3.2 相变

第一版不显式追踪界面，用 `liquid fraction f_l(T)` 表示局部液相比例：

- `f_l = 0` 表示固相
- `f_l = 1` 表示液相
- 在熔点附近设置一个很窄的“mushy zone”来平滑相变

这样可以避免刚开始就处理 Stefan moving boundary。

### 3.3 掺杂输运

掺杂浓度 `C(z,t)` 可写成：

`dC/dt = d/dz[D(T, f_l) * dC/dz]`

其中：

- `D` 在固相很小，在液相很大
- 第一版就用相依赖扩散系数：
  `D = D_solid * (1 - f_l) + D_liquid * f_l`

后续可以再加入：

- 对流项（如果要模拟液相搅动）
- 界面分凝/溶质俘获
- 再凝固界面速度效应

### 3.4 source 层的表示

表面 source 层建议分三档复杂度：

1. `固定表面浓度`
   最简单，适合先验证液相扩散会不会产生合理掺杂深度。

2. `有限库存 surface reservoir`
   用一个面密度 `M_s(t)` 表示表面 source 剩余量，并与硅中表面通量耦合。

3. `显式薄层`
   把 source 作为一个很薄的独立层来解，最真实，但会增加网格和界面处理难度。

建议第一步选第 2 种：`有限库存 reservoir`。它比固定表面浓度真实得多，但实现仍然不难。

### 3.5 当前已知的 source 形态

目前已确认：

- 掺杂元素：`P`
- 表面 source：`PSG (phosphosilicate glass)`

这意味着后续更合理的建模方式是：

- 把 PSG 当作 `有限磷库存`
- 用一个表面库存变量表示可释放的 `P`
- 让 PSG 和熔融 Si 之间通过边界通量交换质量

如果 PSG 对 `532 nm` 的吸收、透过或界面热阻影响不可忽略，热模型也应从“裸硅单层”升级到 `PSG/Si` 双层模型。

## 4. 推荐的开发顺序

### Phase A: 先只做热问题

目标：

- 给定一个激光脉冲
- 计算温度-时间演化
- 提取熔融开始时间、结束时间、最大熔深

输出：

- `T(z, t)` 热图
- `f_l(z, t)` 相图
- `melt depth vs time`

### Phase B: 在已知温度场下做掺杂扩散

目标：

- 用 Phase A 的温度场驱动相依赖扩散
- 加入表面 reservoir
- 得到初版掺杂深度分布

输出：

- `C(z, t)` 热图
- 最终 `C(z)` profile
- 表面 source 剩余量 `M_s(t)`

### Phase C: 热-质弱耦合

如果掺杂浓度不会显著反过来影响吸收和热物性，可以先保持单向耦合：

- 热方程先求
- 扩散方程后求

这通常已经足够覆盖第一版研究。

### Phase D: 升级到 2D 轴对称

当 1D 模型可信后，再加入：

- 高斯光斑横向分布
- 横向导热
- 非均匀熔池形状

这一阶段再切换到更完整的 FEM 会更合适。

## 5. 数值方法建议

### MVP 数值实现

第一版推荐：

- 空间：`1D 网格`
- 时间：`隐式或半隐式`
- 热：`焓法`
- 扩散：`Crank-Nicolson` 或全隐式

这样做的优势：

- 稳定
- 容易扫描参数
- 便于调试相变和边界通量

### 何时必须上 FEM

下面这些情况就值得切换到 FEM：

- 进入 2D/轴对称
- 边界条件变复杂
- 材料层状结构变多
- 要和更复杂的后处理/可视化工具衔接

## 6. 参数分层策略

不要一开始就追求全部真实参数。建议三层：

### Level 0: 演示级

- 常数热导率
- 常数密度
- 简化吸收项
- 固液两档扩散系数

### Level 1: 工程近似

- 温度相关 `k(T)`、`cp(T)`
- 潜热
- 激光脉冲时域函数
- 有限库存 source reservoir

### Level 2: 研究级

- 温度相关反射率/吸收率
- 熔融界面分凝
- 再凝固溶质俘获
- 多脉冲
- 2D 光斑和横向流动

## 7. 可视化建议

第一阶段最值得做的图只有四类：

1. `T(z,t)` 二维热图
2. `f_l(z,t)` 二维相图
3. `melt depth vs time`
4. `final C(z)` 掺杂剖面

第二阶段再做动画：

- 熔融层随时间推进
- 浓度剖面随时间演化

## 8. 项目切分

### Milestone 1

建立 1D 热模型，能输出熔深和熔融时间窗口。

### Milestone 2

在已有温度场上加入掺杂扩散和表面 reservoir。

### Milestone 3

把热模型升级为 `PSG/Si` 双层，并重新驱动扩散。

当前状态：

- 第一版已完成
- 当前默认双层基线下，Si 不熔化，扩散结果为零
- 说明下一轮优先级应转向功率-光斑-fluence 一致性和更真实的光学堆栈

### Milestone 4

补 moving interface、`partition coefficient` / `solute trapping` 等再凝固掺杂物理。

### Milestone 5

可选扩展项：如果后续实验确实需要，再考虑扫描路径和多脉冲工艺效应。

## 9. 每阶段结束后的固定动作

每完成一个阶段，必须额外完成下面四件事：

1. 更新 `docs/phaseX-code-explained.md`
2. 更新 `docs/phaseX-analysis.md`
3. 更新 `docs/formula-reference-register.md`
4. 更新 `docs/session-log.md`

其中 `docs/formula-reference-register.md` 必须明确写出：

- 本阶段真正采用了哪些公式
- 哪些公式仍然只是候选
- 每条公式对应哪篇文献

做参数扫描：

- 激光能量密度
- 脉宽
- reservoir 初始库存
- 液相扩散系数

### Milestone 4

升级到 2D 轴对称 FEM。

### Milestone 5

引入更真实的界面物理，如分凝和再凝固俘获。

## 9. 我建议我们下一步立刻做什么

最合理的第一步不是直接搭完整 FEM，而是先明确以下 6 件事：

1. 激光脉冲形式：单脉冲还是多脉冲
2. 时间尺度：ns、us 还是更长
3. source 类型：P、B、As 或其他
4. source 层是无限供给还是有限库存
5. 我们第一版是否接受 1D
6. 第一版可视化优先看什么：熔深还是最终浓度

如果这些先定下来，我们就可以直接进入 `Milestone 1` 的代码实现。
