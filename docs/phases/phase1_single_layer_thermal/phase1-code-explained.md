# Phase 1 代码解释

本文档解释第一阶段热模型的代码结构与每一段代码的作用。为了保证可读性，我按连续行号的小段来解释；后续如果你希望，我也可以继续细化成更严格的一行一注释版。

## 文件一：`src/laser_doping_sim/phase1_thermal.py`

### 第 1-14 行

- `from __future__ import annotations`：允许在类型注解里更方便地引用尚未定义完成的类型。
- `dataclasses`、`json`、`Path`：分别用于参数结构、结果保存和文件路径处理。
- `matplotlib.use("Agg")`：强制使用无界面后端，这样在命令行或服务器环境里也能直接出图。
- `numpy`：负责数组、向量化计算。
- `scipy.sparse` 和 `spsolve`：用于组装并求解 1D 隐式离散后的稀疏线性系统。

### 第 17-26 行：`MaterialProperties`

- `rho`：硅密度。
- `cp_solid`：固相比热。
- `cp_liquid`：液相比热。
- `k_solid`：固相导热系数。
- `k_liquid`：液相导热系数。
- `latent_heat`：熔化潜热。
- `melt_temp`：熔点。
- `mushy_width`：相变平滑区宽度。

这一组参数定义了“材料本体”的热学行为，也是后面焓法的基础。

### 第 29-35 行：`LaserPulse`

- `fluence`：单位面积总能量，内部单位是 `J/m^2`。
- `pulse_fwhm`：脉冲半高全宽。
- `peak_time`：脉冲峰值出现时间。
- `absorptivity`：吸收率。
- `absorption_depth`：光在硅中的特征吸收深度。

这里把激光脉冲抽象成时间上的高斯脉冲和深度上的指数吸收。

### 第 38-45 行：`Domain1D`

- `thickness`：计算域厚度。
- `nz`：深度方向网格数。
- `dt`：时间步长。
- `t_end`：模拟结束时间。
- `ambient_temp`：初始温度与底部 Dirichlet 边界温度。
- `bottom_bc`：底部边界类型，目前支持 `dirichlet` 和 `neumann`。

### 第 48-58 行：`SimulationResult`

这是一份结果容器，用来统一保存：

- 时间轴
- 深度轴
- 温度场
- 液相分数场
- 熔深曲线
- 激光通量历史
- 本次使用的材料、脉冲和域参数

这样后续 Phase 2 直接复用温度场会很方便。

### 第 61-68 行：`liquid_fraction`

- 第 62-63 行计算固相线和液相线。
- 第 64 行初始化液相分数数组。
- 第 65 行把高于液相线的节点记为完全液相。
- 第 66-67 行把 mushy zone 内的节点线性插值到 `0~1`。

这一步是焓法的核心桥梁之一，因为后面无论是比热还是导热率都要借助 `f_l` 来平滑过渡。

### 第 71-79 行：`apparent_heat_capacity`

- 第 72 行先算液相分数。
- 第 73 行用固液加权形式构造基础比热。
- 第 74-75 行重复相界温度区间。
- 第 76-78 行把潜热折算成 mushy zone 内的一段额外热容。
- 第 79 行返回“表观比热”。

这就是第一阶段采用的“焓法/表观热容法”。优点是实现简单、数值稳定，不必显式跟踪 Stefan 界面。

### 第 82-84 行：`thermal_conductivity`

这里同样通过液相分数在固相导热率和液相导热率之间插值，得到一个随相态变化的有效导热率。

### 第 87-90 行：`gaussian_flux`

- 第 88 行把 FWHM 转换成高斯标准差。
- 第 89 行构造高斯通量的归一化系数，使时间积分等于总 fluence。
- 第 90 行返回任意时刻的入射通量。

这意味着如果把整个时间轴上的 `flux(t)` 积分起来，得到的就是输入的脉冲能量密度。

### 第 93-96 行：`volumetric_heat_source`

- 第 94 行先取某个时刻的表面通量。
- 第 95 行把“表面吸收通量”换成“体热源强度”。
- 第 96 行沿深度施加 Beer-Lambert 指数衰减。

因此第一阶段热源是：

- 时间上高斯
- 深度上指数衰减
- 空间上只考虑法向深度，不考虑横向光斑

### 第 99-168 行：`_assemble_matrix`

这是整个求解器最关键的一段。

#### 第 106-111 行

- 计算节点数与网格间距。
- 计算当前迭代下的体热容系数 `rho * c_eff / dt`。
- 计算节点导热率和界面导热率。

#### 第 113-139 行：底部 `dirichlet`

- 第 114 行把最后一个底部节点从未知量中消元，只求前 `n-1` 个节点。
- 第 115-117 行创建三对角稀疏矩阵的三条对角线。
- 第 118 行构造右端项。
- 第 120-121 行实现表面绝热边界下的首行方程。
- 第 123-131 行逐个组装内部节点。
- 第 130-131 行把底部已知温度对最后一个内部节点的影响移到右端项。
- 第 133-138 行生成稀疏矩阵。

这一版修复了一个重要 bug：之前把底部固定温度直接放进和内部节点同一个大矩阵里，会导致矩阵尺度严重不平衡，细网格时求解器会出现明显病态误差。现在通过消元处理，这个问题已经消失。

#### 第 141-166 行：底部 `neumann`

如果底部也是绝热，则保留全部未知量，并在最后一个节点使用镜像型零通量离散。

#### 第 168 行

如果输入了当前不支持的边界类型，直接报错。

### 第 171-223 行：`run_simulation`

#### 第 178-185 行

- 构建深度网格和时间网格。
- 初始化温度场。
- 初始温度设为环境温度。
- 分配液相分数、熔深和激光通量历史数组。

#### 第 187-204 行

这是时间推进循环。

- 第 188-190 行取上一步温度并生成当前时刻热源。
- 第 192-204 行执行 Picard 迭代：
  - 先用当前猜测温度组装矩阵。
  - 再解隐式线性系统。
  - 如果是底部 Dirichlet，就把已知底温再拼回完整温度向量。
  - 若前后两次温度变化小于容差，则结束当前时间步迭代。

这相当于“隐式时间推进 + 弱非线性迭代”，因为 `c_eff(T)` 和 `k(T)` 都依赖于温度。

#### 第 206-209 行

- 保存当前时刻温度。
- 根据温度求液相分数。
- 找到液相分数超过 `0.5` 的最深节点，把它定义为当前熔深。

#### 第 211 行

补上初始时刻的液相分数，保证输出数组完整。

#### 第 213-223 行

把所有结果打包成 `SimulationResult` 返回。

### 第 226-241 行：`_summary`

这一段从时空结果中抽取几个一眼能读懂的指标：

- 表面峰值温度
- 最大熔深
- 熔融开始时间
- 熔融结束时间
- 最大液相分数
- 脉冲参数和离散规模

这些指标会被写进 `summary.json`。

### 第 244-271 行：`save_outputs`

- 第 248-256 行：把所有数值结果压缩保存到 `phase1_results.npz`。
- 第 258-265 行：把配置参数和摘要指标保存到 `summary.json`。
- 第 267-270 行：调用后处理绘图函数输出四张图。

这样数值数据和图都能保留，便于下一阶段直接复用。

### 第 274-346 行：四个绘图函数

#### 第 274-295 行：温度热图

把 `T(z,t)` 画成二维热图，横轴时间、纵轴深度。

#### 第 298-321 行：液相分数热图

把 `f_l(z,t)` 画成二维图，可以直接看出熔融层的形成、保持和消失。

#### 第 324-333 行：熔深曲线

把熔深随时间变化画出来，是最关键的 Phase 1 输出之一。

#### 第 336-346 行：表面温度曲线

把表面温度与熔点一起画出，方便判断脉冲期间何时进入液相。

## 文件二：`run_phase1.py`

### 第 1-14 行

- 导入命令行、JSON、路径和系统模块。
- 定位工程根目录和 `src` 目录。
- 把 `src` 加入 `sys.path`，这样脚本可以直接导入本地包。

### 第 17-35 行：参数解析器

这里定义了所有可在命令行覆盖的参数：

- 输出目录
- 激光 fluence
- 脉宽
- 峰值时间
- 吸收深度
- 吸收率
- 域厚度
- 网格数
- 时间步长
- 总模拟时间
- 初始温度
- 熔点
- mushy 宽度
- 底边界类型

### 第 38-59 行：构造参数对象

- 先读命令行参数。
- 再把用户输入转换成内部 SI 单位。
- 分别实例化 `MaterialProperties`、`LaserPulse` 和 `Domain1D`。

### 第 61-75 行：执行计算与打印摘要

- 调用 `run_simulation` 执行求解。
- 调用 `save_outputs` 保存数值结果和图。
- 再读取 `summary.json`，把最重要的三个结果打印出来：
  - 峰值表面温度
  - 最大熔深
  - 熔融时间窗口

### 第 79-80 行

这是标准的 Python 命令行入口，让脚本既可以被导入，也可以被直接运行。

## 当前代码结构的优点

1. 物理结构清楚：材料、脉冲、域、结果是分开的。
2. 数值结构清楚：热源、相变、矩阵装配、时间推进分层明显。
3. 便于 Phase 2 复用：后续掺杂扩散可直接读取 `SimulationResult.temperature`。
4. 可调参数清楚：命令行直接可扫参数。

## 当前代码结构的局限

1. 还只是 1D，没加入横向高斯光斑。
2. 光学参数还没按具体波长、温度和表面状态做闭环更新。
3. 相变仍是平滑近似，没有显式界面速度。
4. 还没有掺杂输运、表面 reservoir 和再凝固溶质俘获。
