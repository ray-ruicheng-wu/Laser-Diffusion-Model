# Phase 1 原理分析、可靠性分析与参数表

## 1. 本阶段目标

第一阶段只解决热问题，不求掺杂浓度。目标是：

1. 在给定激光脉冲下计算 `T(z,t)`。
2. 用平滑相变模型得到 `f_l(z,t)`。
3. 提取熔融窗口与熔深曲线。
4. 为下一阶段的液相扩散提供温度历史。

## 2. 模型原理

### 2.1 控制方程

当前模型求解的是一维深度方向热方程：

`rho * c_eff(T) * dT/dt = d/dz (k(T) * dT/dz) + Q(z,t)`

其中：

- `rho` 是密度。
- `c_eff(T)` 是表观热容，包含了相变潜热。
- `k(T)` 是有效导热系数。
- `Q(z,t)` 是激光体热源。

### 2.2 激光热源

第一阶段用的是“时间高斯 + 深度指数衰减”模型：

- 时间上：脉冲通量是高斯函数。
- 深度上：使用 Beer-Lambert 衰减 `exp(-z / delta_abs)`。

这意味着当前模型已经可以描述：

- 短脉冲快速加热
- 近表面优先吸收
- 热量向基底传导

但还没有描述：

- 横向光斑分布
- 多次反射
- 熔池形貌改变引起的吸收变化

### 2.3 相变处理

当前采用的是表观热容法：

- 在熔点附近定义一个 `mushy zone`
- 把潜热等效为一段很大的附加比热
- 不显式追踪固液界面

优点：

- 数值上稳
- 容易和隐式热传导离散耦合
- 很适合做第一阶段原型

代价：

- 固液界面会被平滑
- 很快的界面运动不会被精确追踪

### 2.4 数值离散

当前离散方式是：

- 空间：1D 均匀网格
- 时间：隐式推进
- 非线性：Picard 迭代
- 线性求解：稀疏三对角系统

底部固定温度边界采取了“边界消元”而不是“直接把 Dirichlet 点放入大矩阵”，这是因为细网格下后者会引入明显的数值病态。

## 3. Phase 1 当前结果

默认案例输出位置：

- `outputs/phase1/default_run/`

默认参数：

- fluence: `0.55 J/cm^2`
- pulse FWHM: `10 ns`
- absorptivity: `0.72`
- absorption depth: `80 nm`
- domain thickness: `8 um`
- grid: `500`
- time step: `0.2 ns`

当前默认案例结果：

- 峰值表面温度：`2696.8 K`
- 最大熔深：`400.8 nm`
- 熔融窗口：`27.6 ns` 到 `80.2 ns`

这说明第一阶段热链条已经跑通：激光能量沉积后，表面温度超过熔点，形成一层纳米级液相层，并在几十纳秒时间尺度内重新凝固。

当前输入层还额外记录了两类后续会直接进入 Phase 2 的浓度参数：

- `PSG` 中的 `P` 浓度
- `Si` 基底中的 `Ga` 背景浓度

它们现在只作为已保存输入存在，还没有反向耦合进 Phase 1 的热方程。

## 4. 可靠性分析

这里把可靠性拆成三层：

1. 数值可靠性
2. 物理参数可靠性
3. 模型结构可靠性

### 4.1 数值可靠性

#### 4.1.1 零输入 sanity check

我们已经验证：

- 当 `fluence = 0` 时，模型保持 `300 K`，且不发生虚假熔融。

这一步很重要，因为它说明当前离散格式不会“无中生热”。

#### 4.1.2 网格/时间步敏感性

我们做了三组计算：

| Case | nz | dt (ns) | Peak T (K) | Max Melt Depth (nm) |
| --- | ---: | ---: | ---: | ---: |
| coarse | 300 | 0.30 | 2038.9 | 240.8 |
| baseline | 500 | 0.20 | 1807.0 | 128.3 |
| xfine | 1200 | 0.05 | 1817.2 | 140.1 |

结论：

- `coarse` 明显偏粗，会高估熔深。
- `baseline` 相比 `xfine`：
  - 峰值温度误差约 `0.6%`
  - 最大熔深误差约 `8.4%`

因此当前默认离散对“温度峰值”已经比较稳，对“熔深”属于可用于开发阶段的精度，但还没有到高精度发表级。

#### 4.1.3 数值可信度结论

- 对“有没有熔融、熔融大概持续多久、熔深量级是否在 100 nm 级别”这类问题，当前模型已经可靠。
- 对“熔深精确到几个纳米”这类问题，当前模型还不够，需要更细网格或自适应网格。

### 4.2 基于联网资料的物理可靠性分析

下面的对比采用了三类来源：

1. 硅常温基准性质：CERN Silicon General Properties
2. 硅可见光光学常数：Green 2008
3. 激光硅熔化/掺杂建模：MDPI 2021 Unified Model

#### 4.2.1 热物性

联网资料显示：

- 硅熔点约 `1687 K`
- 室温密度约 `2.328 g/cm^3`
- 室温比热约 `0.67839 J/g/K`
- 室温导热系数约 `1.5 W/cm/K = 150 W/m/K`
- 熔点附近固相导热系数显著下降
- 液相导热系数约 `62 W/m/K`
- 熔化潜热约 `1.802 MJ/kg`

与当前代码对比：

| 参数 | 当前值 | 联网参考 | 评估 |
| --- | ---: | ---: | --- |
| `rho` | 2330 kg/m^3 | 2328 kg/m^3 at RT | 很可靠 |
| `melt_temp` | 1687 K | 1687 K | 很可靠 |
| `cp_solid` | 700 J/kg/K | 678.39 J/kg/K at RT | 很可靠 |
| `latent_heat` | 1.80 MJ/kg | 1.802 MJ/kg | 很可靠 |
| `k_liquid` | 55 W/m/K | 62 W/m/K | 可接受 |
| `k_solid` | 90 W/m/K | 150 W/m/K at RT, 22.8 W/m/K near melting | 中等可靠 |

解释：

- `rho`、`melt_temp`、`cp_solid`、`latent_heat` 这些参数与公开资料一致性很好。
- `k_solid = 90 W/m/K` 不是“某个唯一标准值”，而是一个简化的有效值。因为硅导热系数随温度变化非常强，从室温到熔点前会显著下降。
- 所以当前 `k_solid` 更像“开发阶段的折中常数”，它适合先跑原型，但以后最好改成 `k(T)`。

#### 4.2.2 光学参数

这是当前不确定性最大的部分。

Green 2008 给出的 300 K 本征硅可见光吸收系数表明：

- `390 nm`：吸收深度约 `69.9 nm`
- `450 nm`：吸收深度约 `414.9 nm`
- `530 nm`：吸收深度约 `1273.9 nm`

同时反射率大约为：

- `390 nm`：`0.514`
- `530 nm`：`0.374`

而当前代码默认值是：

- `absorption_depth = 80 nm`
- `absorptivity = 0.72`

这说明：

1. `80 nm` 的吸收深度更接近短波长高吸收情况，而不是典型的 `532 nm` 室温本征硅。
2. `absorptivity = 0.72` 等价于忽略透射时 `reflectivity ≈ 0.28`，这比 532 nm 常温本征硅更“吸收”。

因此，当前热模型在“热学框架”上是可靠的，但在“光学驱动参数”上仍然只是占位默认值。只有当我们把激光波长、表面状态、薄层材料和温度依赖吸收一起锁定后，模型才算真正校准。

#### 4.2.3 与激光掺杂文献的一致性

MDPI 2021 的 unified laser doping model 说明：

- 激光掺杂的关键过程就是“表面熔融 + 熔融硅内扩散 + 再凝固固化掺杂”。
- 对于 `532 nm`、`41-110 ns` 的脉冲条件，文献也是通过求解热方程与扩散方程来做耦合模拟。
- 文献中还指出，对流、表面形貌变化和反射率变化会在更高能量密度下变得重要。

这和我们当前的 Phase 1 路线是一致的：

- 先做纯热问题
- 再做扩散
- 最后再加更复杂的吸收/蒸发/对流机制

也就是说，当前阶段的建模顺序是可靠的；真正还不够可靠的，是尚未指定的工艺参数，而不是建模总思路。

### 4.3 模型结构可靠性

当前模型在哪些方面可靠：

- 能描述纳秒脉冲引起的瞬态升温
- 能描述表层熔化与再凝固
- 能输出 Phase 2 所需的温度历史
- 能支撑参数扫描和后续扩散计算

当前模型在哪些方面还不能宣称可靠：

- 还不能准确预测具体工艺下的绝对熔深，除非补上真实光学参数
- 还不能描述横向熔池形状
- 还不能描述高能量密度下的蒸发、回凝和 Marangoni 对流
- 还不能给出掺杂分布，因为 Phase 2 还没开始

## 6. 当前结论

### 可以认为已经成立的结论

1. 第一阶段 1D 热模型已经可运行且数值上稳定。
2. 焓法 + 隐式求解足以支撑当前阶段。
3. 默认参数下会形成纳米级表层熔融层。
4. 这个温度场已经可以作为第二阶段掺杂扩散的输入。

### 还不能过度相信的结论

1. 当前默认熔深不是某个具体工艺的最终真实值。
2. 光学参数没有锁定波长前，绝对温升预测存在明显不确定性。
3. 高能区下的蒸发与熔池流动目前没被纳入。

## 7. 下一阶段前最重要的参数确认

进入 Phase 2 之前，最值得先锁定的是：

1. 激光脉宽
2. 单脉冲还是多脉冲
3. 表面 source 的具体材料形态
4. 表面 source 是无限供给还是有限库存

## 7.1 已确认的新条件

- 激光波长：`532 nm`
- 重复频率：`500 kHz`
- 掺杂元素：`P`
- 表面 source：`PSG`
- 基底背景掺杂：`Ga`，当前默认 `1.0e16 cm^-3`

## 7.2 基于 532 nm 的重新核对

如果先按“裸晶体硅表面”来处理 532 nm 激光，那么公开光学数据给出的量级大致是：

- 吸收系数 `alpha ≈ 7850 /cm` at `530 nm`
- 吸收深度 `1/alpha ≈ 1.27 um`
- 反射率约 `0.374`
- 吸收率约 `0.626`

这与旧默认值：

- `absorption_depth = 80 nm`
- `absorptivity = 0.72`

差别很大。也就是说，我们之前默认设置把能量沉积得过浅、过集中。

为了做一次可比对的复核，我仅修改光学参数，其余保持默认：

- fluence: `0.55 J/cm^2`
- pulse FWHM: `10 ns`
- absorptivity: `0.626`
- absorption depth: `1274 nm`

复核结果：

- 峰值表面温度：`1294.4 K`
- 最大熔深：`0 nm`
- 结论：`不熔化`

这说明如果采用 532 nm 室温裸硅的常数光学参数，那么我们之前那组会熔化的结果就不再成立。

## 7.3 500 kHz 的意义

`500 kHz` 对应脉冲间隔：

- `1 / f = 2 us`

它本身不能决定单脉冲能量。要确定真正的脉冲能量，还需要至少再知道以下之一：

- 平均功率
- 单脉冲能量
- 光斑尺寸和目标 fluence

关系式是：

- `E_pulse = P_avg / f`
- `fluence = E_pulse / A_spot`

因此，仅凭 `500 kHz` 不能唯一确定“这一脉冲到底有多强”。

## 7.4 500 kHz 下的热积累初步判断

用室温硅参考热扩散率 `0.9 cm^2/s` 估算，`2 us` 内的热扩散长度量级约为：

- `sqrt(alpha * t) ≈ 13.8 um`

这比我们当前 1D 模型中的 `8 um` 深度域还大，说明在“孤立点、无强重叠”的近似下，脉冲间大部分热量有时间扩散掉。

我们把同一单脉冲案例延长到 `2 us` 后也看到了类似现象：

- 表面峰值温度：`1157 K`
- `0.5 us` 时表面温度约：`483 K`
- `1.0 us` 时表面温度约：`363 K`
- `2.0 us` 时表面温度约：`308 K`

所以：

- 如果是“单点单脉冲”或“相邻脉冲重叠不强”，`500 kHz` 本身未必会造成很强的热积累。
- 如果扫描重叠很强，或者平均功率高、光斑小，那么就必须做多脉冲累计模拟。

一旦这些参数进一步确定，下一阶段就可以把扩散方程和 surface reservoir 接上。

## 7.5 PSG 作为 P source 层时，Phase 1 最小接口改动

当前最小且有价值的改动不是把 PSG 直接塞进热方程，而是先把它显式记录成一个源层对象：

- `kind = PSG`
- `dopant = P`
- `notes = source layer placeholder`

这样做的理由是：

1. Phase 1 仍然只负责温度场，不会被过早耦合污染。
2. Phase 2 在接扩散方程时，已经有统一的源层输入对象。
3. 输出文件里会自动保留 PSG/P 的工艺前提，便于后续参数扫描和可追溯性。

结合公开文献，PSG 相关过程通常表现为：

- PSG 层在 POCl3 predeposition 中形成，并且其组成在一定范围内对工艺条件不太敏感。
- PSG 与 Si 之间通常还会存在一层很薄的 SiO2。
- P 常在 PSG/SiO2/Si 接口附近积累。

这意味着下一阶段的扩散模型更合理的边界是“PSG 供源 + 界面有限库存/通量边界”，而不是直接把表面当成裸硅。
